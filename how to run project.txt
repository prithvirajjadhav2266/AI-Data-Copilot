üîß Setup (First Time)

Open project in VS Code

Open the folder Ai_Data_copilot in VS Code.

Create a virtual environment

In the VS Code terminal:

python -m venv .venv


Activate the virtual environment

On Windows (PowerShell):

.venv\Scripts\activate


You should see (.venv) at the beginning of the terminal prompt.

Install dependencies

Make sure you are in the project root (where requirements.txt is), then run:

pip install -r requirements.txt

Configure Gemini API key

The backend expects a Google Gemini key via the environment variable GEMINI_API_KEY. In PowerShell:

$Env:GEMINI_API_KEY = "your-api-key"

Optionally override the model or temperature:

$Env:GEMINI_MODEL = "models/gemini-2.5-flash"
$Env:GEMINI_TEMPERATURE = "0.1"

If you previously set a different model with `setx`, open a fresh PowerShell window after updating the value so the backend sees the change.

ü§ñ LLM Workflows

- **Cleaning Plan**: Use the "Gemini Cleaning Plan" panel in Streamlit to request structured cleaning steps. Dry-run before applying to persist changes.
- **Code Agent**: Use the "Gemini Code Agent" panel for free-form analytics or transformations. The model returns Python `transform(df)` code executed in a restricted backend sandbox. Always dry-run first to inspect the summary and preview.
- All LLM proposals and executions are logged in the backend audit trail for traceability.
